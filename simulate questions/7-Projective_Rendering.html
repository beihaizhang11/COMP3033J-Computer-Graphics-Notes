Answer Key:

Multiple Choice Questions:

1. a - Ray tracing computes one pixel at a time while projective rendering computes multiple pixels

2. a - Translation does not affect surface normals

3. a - Scale → Rotate → Translate (matrices are multiplied in reverse order)

4. b - Determining which objects are visible at each pixel

5. c - Object Coordinate System (OCS)

10. b - The algorithm fails as no correct drawing order exists

True/False Questions:

6. True - Parallel projection preserves parallel lines

7. True - For orthonormal rotation matrices, the inverse equals the transpose

Matching Questions:

8. Translation → Moves object without changing orientation

8. Scaling → Changes size of object

8. Rotation → Changes orientation around axis

9. WCS → Reference frame for entire scene

9. OCS → Define object shape

9. VCS → Camera perspective

Explanations:

1. Ray tracing processes pixels individually by tracing rays, while projective rendering handles multiple pixels by projecting objects.

2. Translation only moves objects without changing their orientation or scale, so normals remain unchanged.

3. Transformation matrices are multiplied in reverse order of the desired operations.

4. Z-buffering solves the visibility problem by keeping track of depth at each pixel.

5. OCS is used to define an object's shape in its local coordinate system.

6. This is a key property of parallel projection used in engineering drawings.

7. This is a special property of orthonormal rotation matrices.

8. Each transformation has distinct effects on object geometry.

9. Each coordinate system serves a specific purpose in the rendering pipeline.

10. This is a classic case where the Painter's Algorithm fails, necessitating other techniques like Z-buffering.
