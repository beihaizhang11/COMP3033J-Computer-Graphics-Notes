Answer Key:

Multiple Choice Questions:

1. b - Converts vertices from Object to World coordinates

2. a - OCS → WCS → VCS → CCS → NDCS → DCS

3. b - Homogeneous coordinates are converted to Cartesian by dividing by w

4. c - When the window is resized

5. b - For double-buffering to prevent visual artifacts

9. c - Pixel Operations

10. b - Easier to parallelize and accelerate

True/False Questions:

6. True - Geometric computations precede rasterization in the pipeline

7. False - The Model Matrix changes more frequently than the Projection Matrix

Matching Question:

8. WCS → Reference frame for entire scene

8. OCS → Defines object shape

8. VCS → Camera perspective

Explanations:

1. The Model Matrix transforms object coordinates to world coordinates, positioning objects in the scene.

2. This is the standard transformation pipeline in computer graphics.

3. Perspective division is a crucial step in converting from homogeneous to Cartesian coordinates.

4. Viewport transformation needs updating when the window size changes to maintain proper display.

5. Double-buffering prevents screen tearing and visual artifacts during rendering.

6. The graphics pipeline performs geometric operations before rasterizing primitives.

7. The Model Matrix changes with object movement, while the Projection Matrix is relatively static.

8. Each coordinate system serves a specific purpose in the rendering pipeline.

9. Fragment tests occur during pixel operations, after rasterization.

10. Projective rendering's main advantage is its suitability for hardware acceleration.
